# Discipline Branches — Gap Option Sets

Each branch below provides the Step 3 options for that discipline. Follow-ups (Step 4) should break each selected gap into 3-4 more specific flavors, following the same multi-select + free-text pattern. See `product-branch.md` for a fully fleshed-out example of how deep this should go for a branch with real calibration behind it.

For Design, Engineering, Sales, and Support: these option sets are reasonable starting points based on general knowledge of what each discipline is evaluated on — they are NOT calibrated from real rejection/feedback history the way the Product branch is. Treat them as a prompt for the person to react to and correct via free text, not as an authoritative diagnostic. Say so plainly if asked.

## DESIGN

**Step 3 options:**
- Solutions feel polished but not grounded in user research/insight
- Doesn't articulate the "why" behind design decisions (defensible rationale)
- Misses systemic thinking (design system, scalability, edge states)
- Doesn't account for engineering feasibility/handoff
- Something else — I'll describe it

**Universal free-text follow-up:** "What did that look like the last time it happened? What was the actual feedback, in their words if you remember it?"

---

## ENGINEERING

**Step 3 options:**
- Solution works but isn't scalable/maintainable long-term
- Doesn't consider trade-offs (performance vs. speed of delivery, etc.)
- Misses edge cases / failure modes
- Communicates in implementation details, not in problem/impact terms
- Something else — I'll describe it

**Universal free-text follow-up:** same as above.

---

## SALES

**Step 3 options:**
- Pitches features, not business outcomes/ROI
- Doesn't handle objections — moves past them instead of resolving them
- Misses economic buyer / decision-making unit
- Lacks urgency or a clear next step in the close
- Something else — I'll describe it

**Universal free-text follow-up:** same as above.

---

## SUPPORT / CUSTOMER SUCCESS

**Step 3 options:**
- Resolves the immediate ticket but doesn't address root cause
- Doesn't tie individual issues back to product/process improvement
- Reactive, not proactive (doesn't flag risk before it becomes churn)
- Struggles to push back on unreasonable asks (internal or customer-facing)
- Something else — I'll describe it

**Universal free-text follow-up:** same as above.
